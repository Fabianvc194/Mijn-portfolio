Operator Tactics

01

Ctrl+Shift+→ is je SQL-mes: in een SELECT-lijst navigeer je razendsnel tussen kolomnamen, aliassen en functies zonder de muis aan te raken. Combineer met Shift voor directe selectie.

02

Box select (Alt+Shift+klik) is goud voor kolom-prefixing: zet cursor boven de eerste JOIN, trek een kolom naar beneden en type een alias — alle regels krijgen tegelijk hetzelfde prefix.

03

Ctrl+D op tabelaliassen: selecteer "tbl1", druk Ctrl+D tot alle matches geselecteerd zijn, hernoem in één keer. Geen Find & Replace-dialoog nodig — nul context-switch.

04

Home → Shift+End → Ctrl+X is de snelste manier om een SQL-regel te verplaatsen. Knip de hele regel, positioneer cursor en plak. Sneller dan Alt+↑/↓ voor grotere afstanden.

05

Win+V (klembordgeschiedenis) is essentieel bij query-iteratie: kopieer meerdere query-varianten, dan kies je precies de juiste versie terug — geen ctrl+z-roulette meer.

06

Ctrl+Alt+↓ (multi-cursor) gecombineerd met Ctrl+Shift+→ selecteert exact dezelfde positie op meerdere regels. Gebruik dit om bij meerdere SELECT-kolommen dezelfde aanpassing te doen.

06

SCRIPT BOILERPLATE · DIRECT KOPIEERBAAR

#Requires -Version 5.1 [CmdletBinding(SupportsShouldProcess)] param( [Parameter(Mandatory)][string]$InputPath, [string]$LogPath = "$PSScriptRoot\logs\$(Get-Date -Format 'yyyyMMdd').log" ) Set-StrictMode -Version Latest $ErrorActionPreference = 'Stop' function Write-Log { param([string]$Message, [string]$Level = 'INFO') $line = "$(Get-Date -Format 'HH:mm:ss') [$Level] $Message" Write-Verbose $line Add-Content -Path $LogPath -Value $line } try { Write-Log "Start verwerking: $InputPath" # uw logica hier Write-Log "Klaar" } catch { Write-Log $_.Exception.Message 'ERROR' Write-Error $_ exit 1 }

Pro Tips

01

-WhatIf op elke destructieve cmdlet (Remove-Item, Stop-Service, Set-ADUser) voordat je uitvoert. Toont exact wat er zou gebeuren — cruciaal in productie-omgevingen.

02

ImportExcel (Install-Module ImportExcel) is de meest onderschatte module. Lees en schrijf Excel zonder dat Office open hoeft — perfect voor geautomatiseerde rapportages.

03

Tee-Object: Get-Process | Tee-Object -FilePath log.txt | Out-GridView — stuurt output tegelijk naar een logbestand én je GUI. Geen keuze tussen zien of opslaan.

04

Zet Set-StrictMode -Version Latest bovenaan elk script. Dit maakt PowerShell strenger: gebruik van ongedefinieerde variabelen geeft meteen een fout — voorkomt stille bugs.

05

Group-Object is de snelste manier om te tellen: Get-EventLog System -Newest 1000 | Group-Object Source | Sort Count -Desc | Select -First 10 — top 10 event-bronnen in één pipeline.

06

Gebruik $PROFILE om aliassen permanent te maken. Voeg toe: Set-Alias grep Select-String, Set-Alias which Get-Command. Laad je profiel opnieuw met . $PROFILE.

Pro Tips

01

Gebruik pathlib.Path in plaats van os.path. Path(__file__).parent / "data" / "file.csv" werkt op Windows én Linux zonder backslash-problemen.

02

print(f"{variabele=}") (Python 3.8+) toont automatisch de naam én waarde: x=42. Sneller dan print(f"x: {x}") — gebruik dit bij tijdelijke debug-prints.

03

logging in plaats van print(): configure eenmalig bovenaan je script. Je kunt het niveau aanpassen (DEBUG/INFO/ERROR) zonder prints te verwijderen — en je hebt automatisch timestamps.

04

df.query("Kolom > 100 and Status == 'Actief'") is leesbaarder dan boolean indexing voor complexe filters. Werkt ook met variabelen: df.query("Waarde > @drempel").

05

Installeer Rich (pip install rich) voor kleurrijke terminal-output, mooie tabellen en progress bars. Eén import, directe verbetering van elke CLI-script die je uitvoert.

06

Gebruik altijd een .venv per project en sla afhankelijkheden op: pip freeze > requirements.txt. Iedereen (inclusief je toekomstige zelf) kan het project daarna exact reproduceren.

BLOKKERING OPSPOREN · DIRECT KOPIEERBAAR

-- Wie blokkeert wie? (plak dit direct bij een incident) SELECT r.session_id AS [SPID], r.blocking_session_id AS [Geblokkeerd_door], r.wait_type, r.wait_time / 1000 AS [Wacht_sec], DB_NAME(r.database_id) AS [Database], SUBSTRING(t.text, (r.statement_start_offset/2)+1, ((CASE r.statement_end_offset WHEN -1 THEN DATALENGTH(t.text) ELSE r.statement_end_offset END - r.statement_start_offset)/2)+1 ) AS [Actieve_Query] FROM sys.dm_exec_requests r CROSS APPLY sys.dm_exec_sql_text(r.sql_handle) t WHERE r.blocking_session_id > 0 ORDER BY r.wait_time DESC;

Pro Tips

01

Ctrl + M altijd aan tijdens performance-onderzoek. Kijk in het uitvoeringsplan naar de dikste pijlen (meeste rijen) en de duurste operaties (% kosten). Table Scan op grote tabel = eerste verdachte.

02

SET STATISTICS IO ON + SET STATISTICS TIME ON samen bovenaan je query. Logical reads is de eerlijkste maatstaf voor query-efficiëntie — niet alleen de uitvoertijd.

03

Gebruik TRY_CAST() in plaats van CAST(). Als een waarde niet te converteren is, geeft CAST een fout die je script stopt. TRY_CAST geeft NULL terug — veiliger in productie-queries.

04

SQL Notebooks (Ctrl+Shift+P → New Notebook): combineer SQL-cellen met Markdown-uitleg. Sla je incidentonderzoeken op als .ipynb en deel ze via Git. Reproduceerbaar en gedocumenteerd tegelijk.

05

dm_db_missing_index_details toont welke indexes SQL Server intern aanbevelenswaardigt. Niet blind volgen, maar bij trage queries is dit de snelste manier om te zien wat er ontbreekt.

06

CROSS APPLY sys.dm_exec_sql_text is de sleutel bij blokkering-analyses. sp_who2 toont de SPID, maar de werkelijke query-tekst haal je op met deze DMV-combinatie — zie het blokkering-snippet hierboven.

PowerShell SQL Outlook

Automatische SQL-rapportage via e-mail

Query SQL, formatteer de uitvoer als HTML-tabel en verstuur automatisch elke ochtend via Outlook — zonder dat je iets hoeft te openen.

  • 01PowerShell maakt verbinding met SQL Server via Invoke-Sqlcmd
  • 02Resultaat omzetten naar HTML-tabel met ConvertTo-Html
  • 03Outlook COM-object aanmaken en mail automatisch verzenden
  • 04Script inplannen via Task Scheduler op vaste tijd
# Dagrapport automatisering $query = "SELECT Naam, Status, Datum FROM dbo.Incidents WHERE CAST(Datum AS DATE) = CAST(GETDATE() AS DATE)" $data = Invoke-Sqlcmd -ServerInstance "SERVER\INSTANCE" -Database "DB" -Query $query $html = $data | ConvertTo-Html -Title "Dagrapport" | Out-String $ol = New-Object -ComObject Outlook.Application $mail = $ol.CreateItem(0) $mail.To = "team@cbs.nl" $mail.Subject = "Dagrapport $(Get-Date -Format 'dd-MM-yyyy')" $mail.HTMLBody = $html $mail.Send()
Besparing: ~15 min/dag

# TOPdesk incident aanmaken via REST API $base = "https://jouwbedrijf.topdesk.net" $cred = [Convert]::ToBase64String( [Text.Encoding]::ASCII.GetBytes("operator:AppPassword123")) $hdrs = @{ Authorization = "Basic $cred"; "Content-Type" = "application/json" } $body = @{ caller = @{ dynamicName = "Jan Janssen" } briefDescription = "Account vergrendeld - Jan Janssen" request = "Gebruiker kan niet inloggen. Account vergrendeld." category = @{ name = "Toegang" } subcategory = @{ name = "Account" } priority = @{ name = "P3" } } | ConvertTo-Json $ticket = Invoke-RestMethod -Uri "$base/api/incidents" -Method POST -Headers $hdrs -Body $body Write-Host "✓ Ticket aangemaakt: $($ticket.number)" -ForegroundColor Green

# Vergrendelde accounts → Slack melding $locked = Search-ADAccount -LockedOut | Select-Object Name, SamAccountName, LockedOut if ($locked) { $tekst = $locked | ForEach-Object { "• $($_.Name) ($($_.SamAccountName))" } # Slack webhook $webhook = "https://hooks.slack.com/services/TXXXX/BXXXX/XXXXX" $bericht = @{ text = "⚠️ *Vergrendelde accounts gevonden*`n" + ($tekst -join "`n") } | ConvertTo-Json Invoke-RestMethod -Uri $webhook -Method POST -Body $bericht -ContentType "application/json" # Optioneel: account direct ontgrendelen # $locked | ForEach-Object { Unlock-ADAccount -Identity $_.SamAccountName } }

Azure Data Studio Power BI SQL

SQL-analyse naar Power BI — volledige keten

Van ruwe data in ADS naar een gepubliceerd Power BI rapport: query bouwen en valideren in ADS, verbinding opzetten in Power BI, en refresh automatiseren. Één keer inrichten, daarna zelfrijdend.

  • 01Query bouwen en testen in ADS — gebruik SET STATISTICS IO ON om performance te valideren vóór je verbindt
  • 02In Power BI: Gegevens ophalen → SQL Server → kies Import of DirectQuery op basis van datavolume
  • 03Sla de SQL-query op als View op de server — Power BI koppelt aan de view, niet aan ruwe SQL
  • 04Gepubliceerd rapport: in Power BI Service → Dataset → Geplande vernieuwing instellen
  • 05Gateway installeren op een machine met netwerktoegang tot SQL Server voor automatische refresh
-- Maak een view aan op SQL Server (ADS) -- Power BI koppelt hieraan — query-wijzigingen hoef je niet opnieuw te verbinden CREATE OR ALTER VIEW dbo.vw_BI_Incidents AS SELECT i.IncidentID, i.Datum, i.Status, i.Prioriteit, m.Naam AS Medewerker, a.Naam AS Afdeling, DATEDIFF(HOUR, i.AangemaaktOp, i.GeslotenOp) AS DoorlooptijdUur FROM dbo.Incidents i JOIN dbo.Medewerkers m ON i.MedewerkerID = m.ID JOIN dbo.Afdelingen a ON m.AfdelingID = a.ID WHERE i.Datum >= DATEADD(YEAR, -1, GETDATE()); -- Power BI connectie: Server = SERVER\INSTANCE, Database = DB, View = dbo.vw_BI_Incidents
Vuistregel: <1M rijen → Import · >1M of real-time → DirectQuery

-- Cel 1: Markdown (Tekst-cel) ## Analyse: Incidentdoorlooptijd per afdeling — april 2026 Doel: onderzoeken welke afdelingen buiten SLA vallen (>4 uur P2). Bron: dbo.Incidents, peildatum 2026-04-19. -- Cel 2: SQL (Code-cel) SELECT a.Naam AS Afdeling, COUNT(*) AS Totaal, SUM(CASE WHEN DoorlooptijdUur > 4 THEN 1 ELSE 0 END) AS BuitenSLA, CAST(SUM(CASE WHEN DoorlooptijdUur > 4 THEN 1.0 ELSE 0 END) / COUNT(*) * 100 AS DECIMAL(5,1)) AS [SLA-breuk %] FROM dbo.vw_BI_Incidents GROUP BY a.Naam ORDER BY [SLA-breuk %] DESC; -- Cel 3: Markdown ## Bevinding Afdeling X heeft 34% SLA-breuk.
